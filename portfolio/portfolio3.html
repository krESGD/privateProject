DB에 상품 관련 정보 insert (MySQL)

상품 정보를 담은 goods table의 각 column 설명

code : 상품을 분류하기 위한 code
소주 : 1*
 일반소주 : 11
 과일소주 : 12
맥주 : 2*
 국산맥주 : 21
 해외맥주 : 22
양주 : 3*
 와인 : 31
 사케 : 32
 보드카 : 33
기타 : 4

name : 상품명

price : 상품의 1개당 가격

stock : 상품의 재고

detail : 상품 상세 설명을 작성한 text

image : 썸네일이나 info 화면에 기본적으로 사용할 상품의 이미지

det_image1 & det_image2 : 상품 상세 설명에 들어가는 image들.
null값을 허용하여 상품 상세 설명에 무조건 2장이 필요하지 않게 하였다.


MyBatis

selectGoodsItem
DB에서 상품 1개에 대한 정보를 가져온 뒤, 가져온 정보들을 상세정보 화면에 각각 삽입한다.

selectGoodsList
DB에서 코드별로 상품들의 리스트 정보를 가져온 뒤,
가져온 정보들을 메인에서 메뉴를 클릭했을 때 나오는 화면에 각각 삽입한다.

selectGoodsCount
DB에서 코드별로 상품들의 갯수 정보를 계산한 뒤,
가져온 정보를 메뉴에서 메뉴를 클릭했을 때 나오는 화면에 삽입한다.

각 메뉴 클릭시 나오는 화면

① : selectGoodsList문을 통해 code에 맞게 나온 상품의 각 정보들을 나열한다.
image를 클릭시 해당 상품의 상세화면으로 이동한다.

② : ④ 클릭시 각 상품별 code를 jsp 및 javascript를 통해 얻도록 하여 해당 code에 맞는 물품명으로 변경해 view에 전달한다.

③ : selectGoodsCount문을 통해 나온 갯수 정보를 해당 위치에 전달한다.

④ : 각 메뉴 클릭시 selectGoodsList문을 통해 code에 맞는 상품 리스트들을 가져오도록 한다.


① ~ ④ 의 mapping 작업 및 버튼 클릭시 동작하는 js. 각 버튼마다 파라미터로 code를 직접 부여했다.


위 화면을 불러오기 위한 Spring Controller

상품 list와 상품의 총 갯수, 그리고 상품 구분을 위한 코드를 각각 얻고 view처리를 해주는 Controller이다.


Service 패턴

개별 상품의 상세화면

①, ② : selectGoodsItem 통해 code에 맞게 나온 상품의 각 정보들을 나열한다.
id parameter는 get 방식을 이용하여 전송했다.
det_image1과 2 및 detail은 상품 상세 정보 창에 전달한다.

JSTL을 이용해 null값이 허용된 det_image1, 2에서 null값이 있는 경우 출력하지 않아도 되도록 코딩하였다.

③ : 해당 code에 맞는 물품명으로 변경해 view에 전달한다.

④ : 현재 페이지 주소를 복사하여 공유할 수 있도록 한다.

Crome 창에서는 주소를 띄워 해당 주소를 복사할 수 있게 했고, 그 외에는 자동으로 Ctrl+C가 되어있도록 구현하였다.


위 화면을 불러오기 위한 Spring Controller
